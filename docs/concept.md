# DOSBox-X Launcher — 项目构想

## 项目概述

一个基于 Python 3.10 + PyQt5 的跨平台桌面应用，作为 DOSBox-X 的图形化前端。
目标是让用户方便地管理 DOS 游戏库，快速配置并启动游戏，无需手动编写 `.conf` 文件。

**技术栈**

- 语言：Python 3.10+
- UI：PyQt5 Widgets
- 数据存储：JSON
- 后端：DOSBox-X（外部进程，用户自行安装）
- 平台：Windows / Linux / macOS

## 功能模块概要

1. **分类管理** — 用户自定义游戏分类（增删改、拖拽排序），始终保留一个不可删除的"全部"分类
2. **游戏库管理** — 添加/删除/编辑游戏信息；删除仅移除条目不删文件
3. **游戏浏览** — 列表视图展示封面、名称、分类、游玩信息；支持搜索过滤和多维度排序
4. **封面图管理** — 支持用户指定本地图片、从截图库选择、或使用默认占位图
5. **截图管理** — 关联 DOSBox-X 截图输出目录，按游戏归属截图，可设为封面
6. **DOSBox-X 配置** — 全局默认配置 + 每游戏差异覆盖的继承机制；内置预设模板
7. **一键启动** — 合并配置、生成临时 conf、调用 subprocess 启动 DOSBox-X、记录游玩数据
8. **全局设置** — DOSBox-X 路径、截图目录、游戏库目录、语言、主题等
9. **其他** — 系统托盘、首次运行向导、游玩记录统计、游戏备注

## 其他要点

- 项目根目录下面存在 `startbox.py` 文件，用于启动项目（尽管是 import dosboxlauncher）；
- 项目的数据文件路径默认保存在 `~/.config/dosboxlauncher` 下面，即数据目录；
- 但是如果项目根目录下面存在 `portable` 目录，则将这个 `portable` 目录作为数据目录；
- 数据目录内有两个文件：`config.json` 用于保存全局配置，`games.json` 用于保存游戏列表；
- 分类信息保存在 `config.json` 里面；
- 软件界面使用全英文，尽管上面的 “界面布局” 里用的中文；

## 项目目录结构

```
DOSBoxLauncher/
  startbox.py               # 启动脚本（import dosboxlauncher 并运行）
  pyproject.toml            # 项目元数据、依赖、工具配置
  AGENTS.md
  LICENSE
  README.md
  portable/                 # 便携模式数据目录（可选；存在时覆盖默认数据路径）
    config.json
    games.json
  dosboxlauncher/           # 主包
    __init__.py
    __main__.py             # 模块入口（python -m dosboxlauncher）
    config.py               # 配置加载/保存
    launcher.py             # DOSBox-X 进程管理
    profiles.py             # 游戏档案模型
    ui/                     # UI 层
  tests/                    # 测试套件
    conftest.py             # 共享 fixtures
    test_config.py
    test_launcher.py
  docs/                     # 文档
    concept.md              # 项目构想
    guide.md                # 项目指南
    prd.md                  # 产品需求文档
    design.md               # 技术设计文档
    spec.md                 # 技术规格说明
```

启动方式有两种：

- `python startbox.py` — 通过根目录启动脚本运行
- `python -m dosboxlauncher` — 通过包的 `__main__.py` 入口运行

两者等价，`startbox.py` 内部即 `import dosboxlauncher` 后调用主入口。
